## 발견 / 결정사항
- **데이터셋 split**: GoPro test = 129장 (train 900장). `summary.md`에 적혀있던 1827/150은 outdated.
- **GoProPairDataset 동작 확인**: resolution=512에서 sample 한 장이 (896, 512)로 64-snap됨
- **diffusion conda env에 누락 의존성**: `einops`, `lpips` 두 개 → 이번 세션에 pip install로 추가 완료
- **기존 SLURM 스크립트의 PROJECT_DIR 버그**: `playground/Motionblur/Motionblur` (빈 디렉토리)로 잘못 가리킴. 신규 SLURM에서는 outer `playground/Motionblur`로 정정
- **Restormer arch 임포트 전략**: basicsr 패키지에 `__init__.py`가 없고 `metrics/__init__.py`가 `niqe`를 끌어와 의존성 충돌. `importlib.util.spec_from_file_location`으로 `restormer_arch.py`만 standalone 로드하면 깔끔
- **SD3.5 ControlNet 디자인 결정**:
  - SD3.5 Large + 공식 Blur ControlNet (`stabilityai/stable-diffusion-3.5-large-controlnet-blur`) 사용
  - cpu_offload + fp16, RTX 3090 24GB에 들어가는 조합
  - 본 baseline은 fine-tuning 없이 zero-shot 성능 측정만 진행 (LoRA fine-tuning은 별도 plan으로 분리)
- **ControlNet 학습 원리 정리**: condition `c = blur img`, target `x_0 = sharp` 페어로 학습하면 deblur ControlNet이 됨. 다만 blur↔sharp 분포 gap, hallucination, 데이터량 문제로 DiffBIR은 Stage1이 분포 gap을 좁힌 뒤 ControlNet에게 photoreal refinement만 시키는 2-stage를 채택. 그래서 본 프로젝트도 (B) 방식으로 진행 결정.
