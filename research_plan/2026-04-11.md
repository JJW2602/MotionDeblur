## 발견 / 결정사항

- **Workspace self-contained 원칙**: 3 workspace 가 각각 400~550 줄씩 독립된 `run.py`. 공통 코드 (`GoProPairDataset`, metric, LoRA train loop) 는 복제되지만 workspace 간 decoupling 이 우선. 공통 module 은 refactor 필요성이 실제로 생기면 나중에 분리.
- **Result 경로 규칙**: `<workspace>/result/<cfg.name>/<mode>/...`. `cfg.name` 에 slash 가 있으면 `Path` 가 자동으로 nested dir 로 해석 (예: `param_tuning/s0.5_g7.0_n40/inference/000007_grid.png`).
- **Hydra 1-combo override 트릭**: test slurm 에서 `"test.strengths=[0.5]"` 형태로 1-element list 를 넘기면 `run.py` 의 internal sweep 이 degenerate 하게 돌아감. `run.py` 수정 없이 SLURM array job 으로 parallelize 가능.
- **SDEdit training objective (결정 유지)**: 기존 코드처럼 sharp 이미지에 SD3 flow-match LoRA (blur 미참여). 분포 편향으로 deblur 효과 기대. 나중에 blur-aware training 으로 바꾸는 것도 옵션.
- **`snap_to_64` 이유**: VAE stride 8 × DiT patch 2 = 16 이 최소 요구치이지만, 일부 ControlNet 의 deeper stride + train dataset 의 64-snap convention 에 맞춰 64 채택. Diffusers pipeline 이 내부에서 16-resize 를 하므로 명시적 snap 이 "조용한 shift" 와 grid 크기 불일치를 방지.
- **ZIP64 주의**: `gopro_deblur.zip` 이 ZIP64 포맷 (v4.5, `ffff ffff` 플래그) 이라 시스템 `unzip` 이 EOCD 를 못 찾음. `python -m zipfile -e` 로 해결. 처음 `unzip` 실패 시점엔 파일이 아직 download 중 (1.4G 상태) 이었고, 완료 후엔 2.0G. 앞으론 이런 큰 zip 은 `python -m zipfile` 을 default 로.
- **tqdm 전략**:
  - Inference = diffusers 내장 pbar (denoising step 한 줄)
  - Test = 내장 pbar OFF + outer (combos) + inner (images, `leave=False`) 2단 nested
  - Train = 커스텀 outer tqdm (`total=max_steps`)
- **test 와 train 의 LoRA checkpoint**: `_load_lora(pipe)` 가 세 모드 모두 공통 진입점. `cfg.checkpoint=null` 이면 no-op (baseline 평가), 경로 지정 시 학습된 LoRA 얹어서 평가.
- **Memory 경로 변경의 부수 효과**: `/scratch2/james2602/research_memory/Motionblur/` 가 이제 empty. 다른 project 들은 여전히 `research_memory/` 를 쓰므로 상위 dir 은 그대로 유지 (SkillStitching 등).
